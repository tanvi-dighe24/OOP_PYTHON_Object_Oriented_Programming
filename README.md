The Four Pillars of OOP
Inheritance: Allows a child class to derive properties and methods from a parent class, promoting code reusability. Python supports single, multiple, and multilevel inheritance.
Encapsulation: Bundles data and methods into a single unit (class) and restricts direct access to some components. Python uses naming conventions like _ (protected) or __ (private) to signal restricted access.
Polymorphism: The ability of different classes to use the same method name but perform different behaviors. It is often implemented through method overriding.
Abstraction: Hides complex implementation details and shows only essential features to the user. In Python, this is typically achieved using the abc module for Abstract Base Classes
